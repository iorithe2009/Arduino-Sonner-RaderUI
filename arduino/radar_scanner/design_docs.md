# Arduino R4 + 超音波センサ + PC レーダー風UI アプリ計画

## 1. 目的
Arduino UNO R4（WiFi / Minima どちらでも可）と超音波センサ（HC-SR04確定）で距離を計測し、
まず Arduino UNO R4 WiFi 内蔵 LED マトリクスで扇状の距離表示を実装し、その後 PC 側でレーダー風のビジュアルUIとしてリアルタイム表示するデスクトップアプリを構築する。

## 2. 全体構成（2フェーズ）
- **Phase 1: ボード単体可視化（先行）**
  - 超音波センサ + SG90 で距離計測
  - Arduino UNO R4 WiFi 内蔵 LED マトリクスに距離帯を扇状（段階表示）で表示
  - シリアルモニタで距離・角度・状態を確認
- **Phase 2: PCレーダー表示（後段）**
  - Arduino と PC を USB 接続
  - PC アプリが COM ポートを購読しデータを取り込む
  - レーダー背景（同心円・扇形・走査線）へ描画

## 3. 推奨技術スタック
### A案（実装容易）
- **Arduino**: Arduino IDE (C/C++)
- **PC UI**: Python + PyQtGraph / PySide6
- **シリアル通信**: pyserial

### B案（高表現・将来拡張）
- **Arduino**: Arduino IDE (C/C++)
- **PC UI**: Electron + WebGL/Canvas
- **シリアル通信**: Web Serial または Node serialport

> まずは A案で PoC を作り、描画表現が不足する場合に B案へ移行する。

## 4. 実装ステップ
1. **ハード接続**
   - HC-SR04 の Trig/Echo を Arduino R4 のデジタルピンへ接続
   - SG90 サーボを PWM ピンに接続
   - 電源と GND 共通化
2. **Phase 1ファームウェア（先行実装）**
   - 超音波パルス送出・エコー時間計測
   - 距離換算（cm）と状態判定（WARNING/NEAR/MID/FAR/OUT）
   - LEDマトリクス表示（警告点滅を含む）
   - シリアルモニタへ距離・角度・状態を出力
3. **Phase 2データ出力拡張**
   - 角度0〜180度往復スキャン
   - PC連携用に `angle,distance,timestamp` をシリアル出力
4. **PC 受信モジュール作成**
   - ポート自動検出
   - パース失敗時の例外処理と再接続
   - データをリングバッファに保持
5. **レーダーUI作成**
   - 背景グリッドと走査線アニメーション
   - 極座標→直交座標変換で点描画
   - 古い点をフェードアウトして残像を演出
6. **統合・調整**
   - 計測周期・表示FPSの最適化
   - ノイズ除去（移動平均 / 外れ値除去）
   - 最小/最大表示距離の調整

## 5. データフォーマット例
### CSV（シンプル）
`<angle_deg>,<distance_cm>,<millis>`

例: `72,134.5,123456`

### JSON（拡張しやすい）
`{"a":72,"d":134.5,"t":123456}`

## 6. UI要件（最低限）
- レーダー表示領域（半円または全円）
- 現在スキャン角の表示
- 検出距離の数値表示
- シリアル接続状態（接続中 / 切断）
- COMポート選択と再接続ボタン

## 7. 性能目標（目安）
- 計測周期: 200ms/ステップ（Phase 1デモ設定）
- UI更新: 30FPS以上
- エンドツーエンド遅延: 200ms以下

## 8. リスクと対策
- **ノイズが多い**: EMA ローパスフィルタ（α=0.3）で平滑化。タイムアウト時は前回値を維持
- **エコー取りこぼし**: タイムアウト設定と無効値処理
- **UIカクつき**: 描画点数の間引き・バッファ制御
- **接続不安定**: 自動再接続・接続監視スレッド導入

## 11. 回路構成（ブレッドボード）
### 推奨ピン割り当て（例）
- **HC-SR04 VCC** → Arduino **5V**
- **HC-SR04 GND** → Arduino **GND**
- **HC-SR04 TRIG** → Arduino **D9**
- **HC-SR04 ECHO** → Arduino **D10**
- **SG90 赤(V+)** → Arduino **5V**
- **SG90 茶(GND)** → Arduino **GND**
- **SG90 橙(SIG)** → Arduino **D6(PWM)**

### ブレッドボード配線手順
1. Arduino の **5V** をブレッドボードの + レールへ接続
2. Arduino の **GND** をブレッドボードの - レールへ接続
3. HC-SR04 の VCC/GND をそれぞれ + / - レールへ接続
4. SG90 の V+/GND をそれぞれ + / - レールへ接続
5. 信号線のみ Arduino の D6, D9, D10 に接続

### 電源を Arduino から取る場合の注意点
- SG90 は起動・加減速時に電流が跳ねるため、**USB給電が弱いとリセットや計測ノイズ**が発生しうる
- まずは以下の条件で開始する
  - サーボ動作角を 10〜170°に制限（端打ち防止）
  - スキャン周期を遅め（例: 20ms〜30ms/step）に設定
  - 必要なら 5V-GND 間に 100µF〜470µF 程度の電解コンデンサを追加
- もし不安定なら、次段階でサーボのみ外部5V化（GND共通）を検討

## 13. 実装済みファイル

### Arduino（Phase 1 + Phase 2 CSV出力）
- ファイル: `arduino/radar_scanner/radar_scanner.ino`
- 対応配線: TRIG=D9, ECHO=D10, SG90信号=D6
- サーボ 10〜170° 往復スキャン（15°/ステップ、STEP_DELAY=200ms）
- 距離計測: EMA ローパスフィルタ（α=0.3）、タイムアウト 12ms
- 距離閾値（デモ用設定）:
  - WARNING: < 10 cm（全行点灯 + 点滅）
  - NEAR: 10〜30 cm（6行点灯）
  - MID: 30〜50 cm（3行点灯）
  - FAR: 50〜100 cm（1行点灯）
  - OUT: 測定不能（0行）
- LEDマトリクス（12列×8行）に角度ゾーン別の距離プロファイルを扇状表示
  - 列 = 角度ゾーン（0〜11）、行数 = 距離帯（下から点灯）
  - 現在スキャン列は縦ライン強調、WARNING 列は 300ms 点滅
- シリアル出力（9600bps）:
  - 人間可読: `角度(deg)  距離(cm)  状態`
  - PC連携用 CSV: `$angle,distance_cm,millis`（OUT時は distance=-1）

### PC アプリ（Phase 2）
- ファイル: `pc_radar/main.py`
- 依存: `pc_radar/requirements.txt`（PySide6, pyqtgraph, pyserial）
- 起動: `pip install -r requirements.txt && python main.py`
- 機能:
  - COMポート選択・接続 / 切断・自動再接続（2秒リトライ）
  - 半円レーダー表示（距離リング・角度線・走査線）
  - 検出点を `FADE_SEC=5秒` でフェードアウト
  - 現在角度・距離をリアルタイム表示（30FPS）
