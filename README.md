# Arduino R4 + 超音波センサ + PC レーダー風UI アプリ計画

## 1. 目的
Arduino UNO R4（WiFi / Minima どちらでも可）と超音波センサ（HC-SR04確定）で距離を計測し、
PC 側でレーダー風のビジュアルUIとしてリアルタイム表示するデスクトップアプリを構築する。

## 2. 全体構成
- **計測層（Arduino）**
  - 超音波センサから距離データを一定周期で取得
  - 必要ならサーボモータで角度を変えながらスキャン
  - `angle,distance,timestamp` のようなフォーマットでシリアル出力
- **通信層（USBシリアル）**
  - Arduino と PC を USB 接続
  - PC アプリが COM ポートを購読しデータを取り込む
- **表示層（PCデスクトップアプリ）**
  - レーダー背景（同心円・扇形・走査線）を描画
  - 受信データを点・軌跡として表示
  - しきい値内の物体をハイライト

## 3. 推奨技術スタック
### A案（実装容易）
- **Arduino**: Arduino IDE (C/C++)
- **PC UI**: Python + PyQtGraph / PySide6
- **シリアル通信**: pyserial

### B案（高表現・将来拡張）
- **Arduino**: Arduino IDE (C/C++)
- **PC UI**: Electron + WebGL/Canvas
- **シリアル通信**: Web Serial または Node serialport

> まずは A案で PoC を作り、描画表現が不足する場合に B案へ移行する。

## 4. 実装ステップ
1. **ハード接続**
   - HC-SR04 の Trig/Echo を Arduino R4 のデジタルピンへ接続
   - （任意）サーボを PWM ピンに接続
   - 電源と GND 共通化
2. **Arduino ファームウェア作成**
   - 超音波パルス送出・エコー時間計測
   - 距離換算（cm）
   - （任意）角度0〜180度往復スキャン
   - シリアルで CSV/JSON 出力
3. **PC 受信モジュール作成**
   - ポート自動検出
   - パース失敗時の例外処理と再接続
   - データをリングバッファに保持
4. **レーダーUI作成**
   - 背景グリッドと走査線アニメーション
   - 極座標→直交座標変換で点描画
   - 古い点をフェードアウトして残像を演出
5. **統合・調整**
   - 計測周期・表示FPSの最適化
   - ノイズ除去（移動平均 / 外れ値除去）
   - 最小/最大表示距離の調整
6. **配布準備**
   - 実行手順書（接続、COM選択、起動方法）
   - Windows向け単体実行化（PyInstaller等）

## 5. データフォーマット例
### CSV（シンプル）
`<angle_deg>,<distance_cm>,<millis>`

例: `72,134.5,123456`

### JSON（拡張しやすい）
`{"a":72,"d":134.5,"t":123456}`

## 6. UI要件（最低限）
- レーダー表示領域（半円または全円）
- 現在スキャン角の表示
- 検出距離の数値表示
- シリアル接続状態（接続中 / 切断）
- COMポート選択と再接続ボタン

## 7. 性能目標（目安）
- 計測周期: 30〜100ms
- UI更新: 30FPS以上
- エンドツーエンド遅延: 200ms以下

## 8. リスクと対策
- **ノイズが多い**: 複数回計測の中央値採用
- **エコー取りこぼし**: タイムアウト設定と無効値処理
- **UIカクつき**: 描画点数の間引き・バッファ制御
- **接続不安定**: 自動再接続・接続監視スレッド導入

## 9. 2週間の進行計画（例）
- **Day 1-2**: 配線・単体計測確認
- **Day 3-4**: Arduino の角度付きデータ送信完成
- **Day 5-7**: PC の受信と基本レーダー描画
- **Day 8-10**: フェード演出・UI操作部・しきい値警告
- **Day 11-12**: ノイズ対策・性能調整
- **Day 13-14**: ドキュメント化・配布準備
